# 중앙을 단일 MCP 서버로 두고 클라이언트는 교체 가능하게

상태: accepted (2026-06-20)

중앙 라우터를 **MCP 서버**(`ask_org` 진입)로 노출한다. 사용자는 웹챗·Slack·자기 MCP 클라이언트(Claude·IDE) 등 무엇으로든 같은 백엔드에 질문하고, 서버가 라우팅·Agent 호출·답변(담당·승인·출처 포함)을 돌려준다. UI는 여러 클라이언트 중 하나일 뿐, 백엔드는 하나다.

## Considered Options

- **MCP 서버**(선택): 사용자가 이미 쓰는 MCP 클라이언트에서 바로 질문 가능, 에이전트 생태계(A2A 포함) 호환. 분산 Agent도 같은 프로토콜로 호출.
- **REST API만**(기각): 자체 클라이언트를 매번 붙여야 하고, 사용자의 기존 도구와 단절.

## Consequences

- 채팅·운영·빌더·처리함·큐 화면은 같은 코어를 웹 백엔드 API로 공유한다.
- 일반 MCP 클라이언트에선 담당·승인 같은 신뢰 표식이 우리 UI가 아니라 텍스트로 노출된다(내용은 보존). 풀 UX 통제가 필요하면 전용 웹챗.
