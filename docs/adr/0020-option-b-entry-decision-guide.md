# 옵션 B(분산) 진입 판단 게이트 — "언제 분산으로 가나"를 세 질문으로 닫는다

상태: accepted (2026-06-22) · **ADR 0017 결정 4의 구체화** — 0017:79가 "후속 결정"으로 남긴 "중앙이 owner 지식을 읽는 것조차 금지가 우리 팀의 진짜 요구인지(→옵션 B 비중)"를 진입 판단 게이트로 닫는다. ADR 0018(커밋 스냅샷 실행)·ADR 0011·0012(분산 전송·백업 워커)와 정합.

## 맥락 — 강등은 깔렸으나 진입 게이트가 없다

ADR 0017이 분산 owner-실행(ADR 0011·0012)을 *기본 경로*에서 **사설 데이터 커넥터 옵션 B**로 강등했고, 그 강등 배너는 SSOT 전반에 깔렸다(CONTEXT Distributed transport 절·각 용어의 "기본 경로 아님" 주석·PRD §6·TRD §2). 하지만 **"언제 옵션 B로 가나"의 판단 기준**은 0017:79가 후속으로 남겼다.

배너만 있고 *진입 판단 게이트*가 없으면, ADR 0017 키스톤이 깨뜨린 거짓 추론 — "owner가 관리하니까 owner가 실행해야 한다" — 로 슬그머니 되돌아갈 길이 열린다. "이 데이터는 owner 사설이다 → 그러니까 분산이다"처럼 *데이터가 사설이라는 사실만으로* 곧장 전체 분산으로 점프하는 오용도 막을 수 없다. 이 ADR이 그 게이트를 명문화한다.

## 결정 — 옵션 B 진입 판단 게이트(세 질문, 순서대로)

새 담당·새 답 시나리오를 설계할 때, 분산을 꺼내기 전에 세 질문을 순서대로 통과시킨다. 앞 질문에서 멈추면 뒤는 보지 않는다.

### 질문 1 — 답이 OKF 번들의 커밋 스냅샷만으로 grounding되나?

YES면 **기본 경로**다. 중앙 `claude -p`가 그 담당 OKF의 최신 커밋 스냅샷을 cwd로 읽어 실행한다(ADR 0017 결정 2·ADR 0018 — `git archive <sha>` 추출 cwd·`Answer.snapshot_sha`). 끝 — **대부분의 케이스가 여기서 멈춘다.** 옵션 B 불필요.

지식이 공유 가능한 마크다운(OKF)으로 표현되는 한, 답은 그 스냅샷을 읽는 것으로 충분하고 분산할 이유가 없다.

### 질문 2 — 답이 owner의 *사설·실시간 데이터/도구*에 의존하나?

질문 1이 NO이고 — 답이 owner의 자기 DB·메일·사내 API·자격증명처럼 *OKF 스냅샷으로는 못 박는 실시간성*에 의존한다면 → **옵션 B-1: 사설 데이터 커넥터**.

owner는 **데이터 접근만** MCP 커넥터로 노출하고, **답 실행은 여전히 중앙**이다 — 중앙이 커넥터로 데이터를 끌어와 grounding한다. "답 생성 전체를 노트북에 보내기"가 *아니다.* 노출하는 건 데이터 접근 한 점뿐이고 실행 위치는 중앙 그대로다.

### 질문 3 — "중앙이 owner 지식을 읽는 것조차" 정책상 금지인가?

규제·기밀·계약상 하드 데이터 경계 때문에 중앙이 owner 데이터를 *읽는 것 자체가* 금지된다면 → **옵션 B-2: 하드 데이터 격리**. owner별 격리 repo + owner 환경 실행으로 간다(이미 구현된 분산 owner-실행 T6.3 / 백업 워커 T6.6 인프라가 바로 이 케이스의 실현이다).

**이 한 케이스에서만 분산이 기술적으로 강제된다.** ADR 0017이 "분산을 강제하는 유일한 요구"로 짚은 바로 그 자리다(0017:13·결정 1의 여덟 능력 중 유일하게 ⭕인 행).

## 근거 — 데이터 격리는 기술적 필연이 아니라 정책 선택

ADR 0017의 핵심을 다시 못박는다: **지식이 공유 가능한 마크다운(OKF)으로 표현되는 한 데이터 격리에 기술적 이득이 없다.** 격리는 정책 선택이지 필연이 아니고, 분산은 가용성(owner PC는 잠든다)·신선도·UX 비용만 든다(0017 근거 "분산의 비용", 백업의 자기모순까지).

그래서 게이트의 무게중심은 한쪽으로 기운다 — **기본은 중앙 실행이고, 옵션 B는 입증 책임이 옵션 B 쪽에 있는 예외다.** 분산으로 가려면 질문 2·3 중 하나를 *적극적으로 입증*해야 하며, 입증이 없으면 질문 1로 돌아간다.

## 오용 방지

- **거짓 추론 차단**: "owner가 관리(거버넌스)하니까 owner가 실행해야 한다"는 ADR 0017 키스톤이 깨뜨린 끊어진 고리다. 관리의 일곱 거버넌스 능력(정의·지식·범위·검토·합의·귀속·라이프사이클)은 모두 중앙 실행과 양립한다(0017 결정 1). 관리 주체라는 사실이 실행 위치를 분산으로 끌고 가지 않는다.
- **B-1과 B-2를 혼동 금지**: B-1(데이터 의존)은 *데이터 접근만* 노출하고 실행은 중앙에 남는다. B-2(하드 격리)만 *전체 격리 실행*이다. **데이터가 사설이라는 사실만으로 곧장 전체 분산(B-2)으로 점프하지 말 것** — 사설 데이터는 질문 2(B-1)에서 먼저 걸러지고, 질문 3(정책상 읽기 금지)을 통과해야만 B-2다.

## Consequences

- **불변식 영향 없음** — 미아 없음·Authority 중앙·전이≠기록·등록 무결성·노출 불변식 모두 무관하다(순수 문서·판단 가이드, 새 코드 0). 게이트는 *설계 시점의 분기 판단*이지 런타임 전이가 아니다.
- **갱신 대상**: PRD §6(분산 전송 옵션 B 항목에 게이트 요약)·TRD §2(분산 owner-실행 단락에 게이트 참조)·CONTEXT Distributed transport 재포지셔닝 배너(게이트 3질문 + B-1/B-2 구분)·ADR 0017:79 미해결 항목(이 ADR로 닫힘 표시)·tasks T7.5.

## ADR 0017과의 관계

이 ADR은 ADR 0017 결정 4(분산 = 사설 데이터 커넥터 옵션으로 강등)를 *구체화*하고, **0017:79가 "진짜 케이스 판단"으로 남긴 미해결을 닫는다.** 0017:79의 다른 미해결은 각자 다른 결정이 닫거나 예정이다 — 변경 전파의 정확한 알고리즘은 ADR 0019가 닫았고, SSO 공급자 선택은 T7.1이 닫는다.
